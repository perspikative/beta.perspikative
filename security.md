# Security Policy

_Last updated: July 2, 2026_

## Supported Versions

The latest version of Perspikative is actively maintained and receives security updates when necessary.

Older versions may no longer receive security fixes.

| Version | Supported |
|----------|-----------|
| Latest   | ✅ Yes |
| Older releases | ❌ No |

Please note that this repository is the beta version of [this repository](https://github.com/perspikative/perspikative) and may contain mistakes and experimentaions.

---

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability in Perspikative, please report it responsibly.

**Please do not disclose security vulnerabilities publicly before they have been investigated and resolved.**

When reporting a vulnerability, please include as much information as possible:

- A clear description of the issue.
- Steps to reproduce the vulnerability.
- The affected page(s) or feature(s).
- Screenshots or proof-of-concept (if applicable).
- Any suggestions for a possible fix.

---

## Responsible Disclosure

We appreciate responsible security research.

Please:

- Give us reasonable time to investigate and resolve the issue.
- Avoid publicly sharing exploit details before a fix is available.
- Do not access, modify, or delete data that does not belong to you.
- Do not disrupt the availability of the website or its services.
- Do not perform denial-of-service (DoS/DDoS) attacks.
- Do not exploit vulnerabilities beyond what is necessary to demonstrate their existence.

Good-faith security research is appreciated and helps improve Perspikative.

---

## Out of Scope

The following are generally considered out of scope:

- Social engineering attacks.
- Spam reports.
- Vulnerabilities requiring physical access.
- Issues caused by outdated browsers or unsupported software.
- Self-XSS requiring users to paste code into their own browser.
- Reports without sufficient information to reproduce the issue.

---

## Security Updates

Confirmed vulnerabilities will be investigated and fixed as soon as reasonably possible.

The severity of the issue will determine the priority of the fix.

---

## Contact

If you need to report a security issue, please contact the project owner through the official Perspikative contact channels.

Thank you for helping keep Perspikative safe for everyone.
